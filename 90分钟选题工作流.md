# 90分钟选题工作流

## 目标

每天用 90 分钟，稳定完成下面三件事：

1. 找到最近 3 天内值得讲的真实信号
2. 从这些信号里筛出 1 到 3 个候选题
3. 至少确定 1 个当天最值得做的题

核心原则只有一句：

不是追热点，而是用最近 3 天的新信号，持续喂养已经建立起来的认知主线。

## 四类母题

以后所有选题，优先只往下面四类里挂：

1. AI软件工程
2. AI时代的人与社会认知
3. AI技术原理与系统解释
4. 工具、产品与开源实践观察

如果一条信息挂不到这四类里的任何一类，大概率不值得做。

---

## 第一阶段：扫信号

时间建议：25 分钟

目标不是看很多，而是快速建立当天的信号面。

默认只扫以下五类内容：

1. X
2. arXiv
3. YouTube
4. 公众号
5. 官方博客 / 项目主页

### 1. X 怎么扫

X 的作用，不是拿来判断最终选题，而是拿来看最近 3 天海外 AI 圈到底在集中讨论什么、争议什么、误判什么。

重点不是刷首页，而是固定看一小批长期有效的人。

优先关注下面几类账号：

- 模型公司官方：
  `@OpenAI`
  `@AnthropicAI`
  `@GoogleDeepMind`
  `@xai`
  `@MistralAI`
- 产品和开发者生态官方：
  `@LangChainAI`
  `@llama_index`
  `@huggingface`
  `@vercel`
  `@cursor_ai`
- 有一线工程判断的人：
  `@karpathy`
  `@ylecun`
  `@swyx`
  `@jxnlco`
  `@dsp_`
  `@lateinteraction`
  `@rasbt`
- 做 Agent / 编码 / 工程化的人：
  `@bentossell`
  `@scobleizer`
  `@jerryjliu0`
  `@hwchase17`
  `@sydneyrunkle`
- 做 AI 社会观察与认知判断的人：
  `@pakpakchicken`
  `@nearcyan`
  `@morethanmoore`

X 的具体扫法：

- 先看这些账号最近 3 天发了什么
- 再看他们在转发什么，评论区在争什么
- 只记录高频重复出现的概念、产品、论文、争议
- 不要因为某条内容爆了就默认它值得讲

在 X 上每扫到一条有价值的信息，只记一句：

- 发生了什么
- 谁在讨论
- 争议点是什么

例子：

- Anthropic 发了新模型，大家讨论点不只是更强，而是更敢放权
- LangChain 发了 async subagents，讨论点是中途 steering 而不是并行本身

### 2. arXiv 怎么扫

arXiv 的作用，是看最近 3 天有没有真正能改写认知框架的新论文。

你不需要每天大量读论文，只需要做“方向筛查”。

优先看这几个分类：

- `cs.AI`
- `cs.CL`
- `cs.LG`
- `cs.HC`

具体扫法：

- 只看最近 3 天提交的论文
- 先看标题和摘要，不要一上来读全文
- 看它是不是在回答一个值得长期讲的问题，而不是只是 benchmark 提升
- 优先保留以下几类论文：
  - 改写旧共识
  - 给出新机制解释
  - 说明能力边界或失败条件
  - 能和产品、工程、人的变化连起来

默认不优先的论文：

- 只是又提升了几个点
- 只是更多数据、更多参数、更多 tricks
- 很难转成普通人能理解的认知内容

你在 arXiv 上每扫到一篇，先只记四件事：

- 这篇论文解决的到底是什么问题
- 它和过去的做法有什么本质区别
- 它最有价值的判断是什么
- 它能挂到哪条母题上

### 3. YouTube 怎么扫

YouTube 的作用，不是看娱乐信息，而是看哪些话题已经开始被系统化讲述。

它能帮你判断：

- 一个话题是不是已经从零散讨论进入成型阶段
- 海外创作者是从什么角度在讲
- 用户真正爱问的问题是什么

优先看这几类频道：

- AI 新闻与产品动态：
  `Matthew Berman`
  `The AI Breakdown`
  `AI Explained`
  `Latent Space`
  `Fireship`
- 工程与实战：
  `CodeRabbit`
  `LangChain`
  `Weights & Biases`
  `Hugging Face`
  `IBM Technology`
- 研究与原理解释：
  `Yannic Kilcher`
  `Two Minute Papers`
  `DeepLearningAI`

YouTube 的具体扫法：

- 只看最近 3 天上传的视频
- 不必完整看，先看标题、封面、简介和评论区
- 如果一个主题连续被多个频道讲，说明它可能值得进入候选池
- 如果一个主题播放量不一定高，但评论区高频出现同一个问题，也值得记

如果你当天没有明确想看的频道，也可以直接搜：

- `AI agents`
- `Claude Code`
- `Harness Engineering`
- `browser automation AI`
- `context engineering`
- `world models`

搜索后只看最近 3 天，并优先看：

- 标题是否集中在同一个判断上
- 不同创作者是不是在重复同一个误解

### 4. 公众号怎么扫

公众号的作用，是看中文语境下哪些话题已经开始形成自己的论证框架。

它的价值不在新闻，而在：

- 中文圈已经怎么理解这个东西
- 大家讲浅了什么
- 哪些中文表达方式值得借用

具体扫法：

- 只看最近 3 天的文章
- 优先搜你当天在 X / arXiv / YouTube 里看到的高频词
- 重点看：
  - 标题在怎么下判断
  - 开头把问题立成了什么样
  - 中文作者有没有只停留在资讯层

公众号里优先记录两种内容：

- 一个中文作者给了比海外更适合中文表达的结构
- 一个中文作者把真正该讲的地方讲浅了，这正好给你反向切口

### 5. 官方博客 / 项目主页怎么扫

官方博客和项目主页的作用，是拿一手信息，避免全靠二手解读。

默认优先看这些来源：

- OpenAI
- Anthropic
- Google / Google DeepMind
- LangChain
- Hugging Face
- Vercel
- Cloudflare
- GitHub
- 重要开源项目主页和 GitHub release notes

具体扫法：

- 只看最近 3 天的新发布
- 先看标题、发布日期、核心更新点
- 再看这次更新到底改变了什么工作流，而不是只看多了什么功能
- 如果是开源项目，再顺手看 release notes 和 issue 讨论

你在官方源上重点找三种信号：

- 新能力
- 新限制
- 新范式

### 第一阶段的输出格式

只写一份简短信号清单，不要展开写作。

```md
## 今日信号
- 
- 
- 
- 
- 
```

## 第二阶段：统一过 6 个问题

时间建议：20 分钟

这一阶段是强制动作。

每扫到一条信息，不要先想着它火不火，也不要先想着怎么写。必须立刻统一过下面 6 个问题，并记录下来。

### 六个问题

1. 这条信息属于哪一类母题  
   只能挂在下面四类之一：
   `AI软件工程`
   `AI时代的人与社会认知`
   `AI技术原理与系统解释`
   `工具、产品与开源实践观察`

2. 它为什么是现在值得讲，而不是以后再讲

3. 它在纠正什么误解，或者暴露了什么认知空白

4. 它背后真正的问题是什么  
   不要停留在表面新闻，要问自己本质矛盾在哪

5. 它是短期热点，还是可以延展成长期系列

6. 我能不能讲出和别人不同的解释角度  
   如果只能重复已有总结，先不要做

### 第二阶段的记录格式

每条信息统一按下面格式记：

```md
### 信息 1
- 母题：
- 为什么现在值得讲：
- 它纠正了什么误解：
- 背后真正的问题：
- 是热点还是系列：
- 我的独特解释角度：
```

只有在这 6 个问题过完之后，这条信息才有资格进入候选题。

## 第三阶段：转成我的题目

时间建议：15 分钟

这一阶段的目标，不是重复新闻标题，而是把信息转成你自己的认知型题目。

固定使用下面 5 个句式改写：

1. 真正值得讲的不是 X，而是 Y
2. 大家以为问题在 X，其实问题在 Y
3. 这件事真正改写的不是 X，而是 Y
4. 很多人把 X 当能力，其实它更像是 Y
5. 如果你只看到 X，你就会错过 Y

改写要求：

- 不要写成资讯标题
- 不要只复述产品名、论文名或发布时间
- 要把标题直接变成一个判断
- 标题最好一眼看出冲突点或认知反转

例子：

- 不写：
  `LangChain 发布 async subagents`
- 改成：
  `多智能体真正难的不是并行，而是任务做一半时你还能改主意`

- 不写：
  `OpenMobile 论文发布`
- 改成：
  `开源移动智能体追不上闭源，真正缺的不是模型，而是错误恢复数据`

### 第三阶段的记录格式

```md
## 候选题
- 
- 
- 
```

默认保留 1 到 3 个候选题，不要太多。

---

## 第四阶段：做素材卡

时间建议：20 分钟

这一阶段的目标，是把候选题从一个想法，变成可写、可讲、可复用的半成品。

每个候选题都必须做一张素材卡。

### 素材卡必须包含的内容

1. 题目
2. 母题
3. 时间窗口  
   默认写最近 3 天的时间范围
4. 信号来源  
   必须写平台名、原始链接、时间
5. 事件层  
   到底发生了什么
6. 判断层  
   你真正想讲的是什么
7. 可讲切口  
   至少列 2 到 3 个
8. 可延展下一题  
   这题后面还能接什么系列

### 第四阶段的素材卡模板

```md
### 题目

- 母题：
- 时间窗口：
- 信号来源：
  - 平台 / 链接 / 时间
  - 平台 / 链接 / 时间
- 事件层：
- 判断层：
- 可讲切口：
  - 
  - 
  - 
- 可延展下一题：
  - 
  - 
```

素材卡做完的标准：

- 你明天不重新搜索，也能直接开始写
- 你知道这题属于什么母题
- 你知道这题的判断，不只是知道“发生了什么”

---

## 第五阶段：收口 10 分钟

时间建议：10 分钟

这一阶段必须做判断，不允许只停留在搜集。

每天最后，统一写三行：

```md
## 今日结论
- 今日最值得做：
- 进入备选池：
- 今日不做：
```

### 收口标准

1. 今日最值得做  
   只能有 1 个  
   这是你当天如果只做一个题，最该做的那个

2. 进入备选池  
   写 1 到 2 个  
   这些题今天不写，但后面能接着做

3. 今日不做  
   明确写出为什么不做  
   常见原因包括：
   - 只有新闻，没有判断
   - 只能复述，讲不出新切口
   - 挂不到四类母题上
   - 信息不够扎实

这一阶段的意义，是逼自己从“我看到了很多”进入“我真的做了选择”。
