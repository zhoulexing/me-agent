# 信息记录标准

## 📝 通用模板

```markdown
---
title: [名称]
category: [分类]
date: [YYYY-MM-DD]
tags: [标签1, 标签2]
---

## [是什么]
[一段话描述：这是什么/在哪里/做什么]

## [核心信息]
- **链接/地址**: [文档链接 OR 实际地址]
- **关键点**: [2-3个要点]
  - [要点1]
  - [要点2]
  - [要点3]

## [为什么记录]
[为什么值得记录/注意事项/后续计划]
```

---

## 🏷️ 分类

| 分类代码 | 适用场景 | 示例 |
|---------|---------|------|
| experience | 体验类 | 去过的好地方、餐厅、活动 |
| doc | 文档类 | 教程、安装指南、操作文档 |
| other | 其他 | 备忘、临时信息 |

---

## 📌 示例

### 示例1：体验类（去过的地方）

```markdown
---
title: 三里屯某咖啡馆
category: experience
date: 2026-03-22
tags: 咖啡馆, 安静, 适合工作
---

## 是什么
三里屯附近的一家安静咖啡馆，适合工作或约会，环境舒适人不多。

## 核心信息
- **地址**: 三里屯路XX号，距地铁10号线X口5分钟
- **关键点**:
  - 有插座和WiFi
  - 人少不吵
  - 咖啡性价比高

## 为什么记录
下次想再来办公，记得带上电脑，工作日下午人最少。
```

### 示例2：文档类（安装教程）

```markdown
---
title: OpenClaw 安装指南
category: doc
date: 2026-03-22
tags: 开发工具, 安装, Claude
---

## 是什么
OpenClaw 的官方安装文档，用于配置 Claude 相关开发环境。

## 核心信息
- **文档链接**: https://example.com/openclaw-install
- **关键点**:
  - 需要 Node.js 18+
  - 安装命令：`npm install -g openclaw`
  - 配置文件在 `~/.openclaw/config.json`

## 为什么记录
下次重装电脑或给同事配置时直接查这个，节省时间。
```

### 示例3：文档类（快捷操作）

```markdown
---
title: Claude Code 快捷键
category: doc
date: 2026-03-22
tags: 快捷键, Claude, 效率
---

## 是什么
Claude Code 常用快捷键操作汇总。

## 核心信息
- **文档链接**: [官方文档链接]
- **关键点**:
  - `Cmd + Enter`: 发送消息
  - `Cmd + K`: 打开对话
  - `/commit`: 快速提交

## 为什么记录
刚开始用经常忘记，记下来方便随时查。
```

---

## 📁 命名规范

`YYYY-MM-DD-[分类]-[名称].md`

例如：
- `2026-03-22-experience-三里屯咖啡馆.md`
- `2026-03-22-doc-OpenClaw安装.md`
- `2026-03-22-doc-Claude快捷键.md`
