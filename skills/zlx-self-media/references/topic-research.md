# 选题研究

## 目标

这个 reference 用于持续做 AI 相关选题研究，不是一次性头脑风暴。

核心任务：

1. 通过联网搜索，判断最近 3 天内什么话题值得讲
2. 把外部平台信号组织进稳定的 AI 内容框架里
3. 对每个候选题判断它值不值得讲，而不是只罗列热点

深度研究的要求：

- 不只看什么火，还要看为什么火
- 不只看标题，还要看谁在讨论、争议点是什么、用户卡在哪
- 不只判断有没有流量，还要判断有没有长期解释价值

如果最近 3 天内没有搜到足够有效的信息，必须明确提示找不到，不允许拿更早的内容冒充最近信号。

## 素材源

1. arXiv

- YouTube：必须优先基于 `yt-dlp` 搜索，不能只凭记忆给频道或视频；重点看最近视频标题、主题集中度、讨论角度和高频问题

- 微信公众号：重点看中文语境下是否已经开始深度讨论，哪些论证框架值得吸收

## 内容范围

底层原则：只做跟 AI 相关的内容。

默认只围绕以下四类展开：

1. AI时代的人与社会认知
2. AI软件工程
3. AI技术原理与系统解释
4. 工具、产品与开源实践观察

四类说明：

### 1. AI时代的人与社会认知

适合讲：

- AI 时代人的能力结构如何变化
- 学习方式、职业判断、教育方式、创造力和注意力如何变化
- 企业、组织、行业在 AI 时代的摩擦、错位和重构

### 2. AI软件工程

适合讲：

- Agent
- Context Engineering
- Harness Engineering
- AI 工作流设计
- AI 能力如何接入真实软件系统和生产流程

### 3. AI技术原理与系统解释

适合讲：

- AI 模型或系统是怎么运作的
- 为什么某个新范式开始成立
- 一个能力的机制、边界、代价和失败条件

### 4. 工具、产品与开源实践观察

适合讲：

- AI 工具怎么用、值不值得用
- 产品设计和使用体验
- 开源项目、框架和工程实践观察

## 内容要求

时间要求：

- 默认只看最近 3 天的内容信号
- 输出时必须写清素材时间
- 如果最近 3 天搜不到足够内容，必须明确说明“最近 3 天没有找到足够值得讲的内容”

选题判断要求：

- 必须和 AI 明确相关
- 必须能回答为什么现在值得讲
- 必须能说明它属于上面四类中的哪一类
- 必须能判断这个题是短期热点，还是可以延展成长期系列
- 必须避免只做表层新闻复述

优先保留的题：

- 在改写一个旧共识
- 在解释一个新范式为什么成立
- 同时给出能力边界、失败条件或成本拐点
- 能从研究、产品、工作流一路讲到人的判断和组织变化
- 适合扩成系列，而不是只能做一次热点解读

默认不优先的题：

- 只是又一个 benchmark 提升
- 只是堆模型、堆数据、堆 tricks
- 只有结论，没有机制
- 迁移价值弱，只对作者自己的设定成立
- 在多个平台已经被讲烂，没有新的解释角度

种子题库参考：

- AI时代的人与社会认知：
  `在AI时代，该如何衡量工时？`
  `AI时代人类注意力与创造力培养探讨`
  `AI时代计算机专业学习的价值与思维培养`
  `人人都能coding,软件会消失吗？`

- AI软件工程：
  `从推理思维到智能体思维`
  `AI放权，前提是有一个值得放权的框架`
  `Harness Engineering到底是什么？`
  `CLI和SKILL在设计时如何处理超大数据量问题`
  `从 SKILL 的设计到工具协同，再到 Context Engineering`
  `当具备技能的单智能体可以替代多智能体系统，以及它们何时会失败`

- AI技术原理与系统解释：
  `AI是如何生成图片的`
  `一文彻底了解浏览器自动化，cdp、playwright、browser-user、midscene、browsermcp`
  `世界模型到底是什么？以及谷歌、李飞飞、阿里三条路线的区别`
  `面向大数据量查询的分层缓存策略：从条件查询到 Parquet + DuckDB`

## 内容输出

输出选题时，每个候选题必须包含以下信息：

1. 选题标题
2. 所属分类
3. 资料来源
4. 为什么值得讲

资料来源必须包含：

- 平台名称
- 原始链接
- 素材发布时间或检索时间

“为什么值得讲”至少回答这几个问题：

- 它为什么是现在值得讲，而不是以后再讲
- 它解决了什么认知空白、信息差或现实问题
- 它是热点跟进，还是可以沉淀成长期系列
- 它和已有内容相比，新的切口在哪里

默认输出格式：

### 选题标题

- 分类：
- 资料来源：
- 为什么值得讲：

## 约束

- 不要只靠记忆给题，必须联网搜索
- 不要只给平台热点列表，必须做判断和筛选
- 不要脱离 AI 主线发散到泛科技或泛成长
- 不要拿超过最近 3 天的旧内容冒充最近趋势
- 不要把没有原始链接和时间的信息当作有效资料源
- 如果找不到足够信息，要明确说找不到，不要硬编
- 选题表达不要有 AI 味太重的空话，结论要具体
