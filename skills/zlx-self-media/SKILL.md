# Self Media

## Overview
这个 skill 用于处理自媒体相关任务，重点不是泛泛聊内容，而是帮助用户持续找到值得做的题。

当前能力：
1. **选题研究**：基于联网搜索做多平台信号收集，围绕四类内容体系输出选题
2. **内容写作**：在用户确认选题后，先给结构和逻辑势能选项，再按要求写作
3. **标题优化**：基于选题或正文生成更能传播的标题
4. **视频剪辑包装**：给本地口播视频加字幕、插入信息图卡片、做基础音画优化，输出可发布 MP4
5. **小红书视频发布**：通过 browser-use 将本地视频发布到小红书创作者平台
6. **抖音视频发布**：通过 browser-use 将本地视频发布到抖音创作者平台
7. **公众号稿整理**：基于已有文字稿整理成公众号文章，尽量保留原文内容，并按需要生成配图和公众号 Markdown

## Workflows

### 1. 选题
- 先读取 `references/topic-research.md`
- 必须联网搜索，不允许只靠记忆直接给题
- 必须进行深度研究，不只看表面热点，还要判断背后的共性问题、讨论焦点和可讲深度
- 默认检索平台：小红书、抖音、公众号、X、arXiv
- 先看平台信号，再回到四类框架里组织选题

### 3. 写作
- 先读取 `references/writing-principles.md`
- 必须先确定选题，未确认前不要直接开写
- 选题确认后，先基于内容给出适合的结构选项和逻辑势能选项
- 等用户选定后，再开始正式写作
- 写完后再读取 `references/title-principles.md` 生成标题

### 4. 公众号稿整理
- 先读取 `references/wechat-article-from-draft.md`
- 适用于用户已经给出文章、口播稿、笔记、技术认知稿，要求整理成公众号文章
- 核心原则是保留原文表达，除非用户明确要求重写
- 如果需要配图，先基于内容节点生成少量配图提示词，再生成图片并插入到固定位置
- 最终 Markdown 必须适配公众号编辑器，避免过度技术文档化

### 5. 标题
- 读取 `references/title-principles.md`
- 基于已有主题或文稿生成标题，不要脱离内容胡乱起标题

### 6. 视频剪辑包装
- 先读取 `references/video-editing-standard.md`
- 适用于用户给出本地视频，要求剪辑、加字幕、插入图片/卡片、做成可发布视频
- 默认不改动原始视频，在工作区生成 `video_edit_work/` 中间产物和 output 成片
- 对已有口播视频，优先使用 `ffmpeg + faster-whisper + Python/Pillow` 标准流程
- 只有用户明确要复杂动态图文、标题动画、重点词高亮、宣传片式包装时，才使用 HyperFrames
- 输出前必须抽帧检查普通字幕段、卡片段和结尾段，确认字幕不越界、卡片不裁切、没有黑屏

### 7. 小红书视频发布
- 先读取 `references/xiaohongshu-video-post.md`
- 默认 headless 模式，使用 `--profile "用户1"` 复用登录状态
- 按流程操作：打开创作平台 → 上传视频 → 填写标题和正文 → 截图确认 → 发布
- 正文默认只包含两个固定标签：`#人工智能` `#视频日记`
- 发布前必须截图给用户确认，确认后才能点击发布
- 操作完成后关闭浏览器

### 8. 抖音视频发布
- 先读取 `references/douyin-video-post.md`
- 默认 headless 模式，使用 `--profile "用户1"` 复用登录状态
- 按流程操作：打开创作者平台 → 上传视频 → 填写标题和正文 → 截图确认 → 发布
- 正文默认只包含两个固定标签：`#人工智能` `#视频日记`
- 发布前必须截图给用户确认，确认后才能点击发布
- 操作完成后关闭浏览器

## Constraints
- 不要 AI 味太重
- 选题必须基于真实平台信号，不要凭空编趋势
- 选题研究不能停留在浅层热点罗列，必须做深度研究和交叉判断
- 涉及“最近、最新、现在流行什么”时，必须给出具体平台和时间判断
- 选题优先围绕四类内容体系，不要重新发散成很多平行方向
- 写作必须有对象感，默认直接对你说话
- 幽默感可选，但不能为了幽默破坏清晰度
- 结构和逻辑势能要分开判断
- 默认不要使用引号和破折号，除非用户明确要求
