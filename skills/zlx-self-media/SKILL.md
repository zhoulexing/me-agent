# self-media

## 概述

这个 skill 支持两个可以独立使用的能力：

1. 把 `06-流水账/` 中用户选定的内容整理成口播视频文字稿。
2. 基于现成的文字稿或文章草稿生成候选标题。

只执行用户明确需要的能力。如果两个都要，按“先写稿、再出标题”的顺序处理。

## 能力一：流水账生成口播稿

按下面的流程执行：

1. 读取用户指定的 `06-流水账/` 文件或用户贴出的内容。
2. 判断用户想使用哪些条目；如果用户没有说得很明确，就根据提问和内容列表推断最可能的主题。
3. 从 `02-个人认知/` 和 `03-技术认知/` 中读取 1 到 3 篇最相关的风格参考。
4. 按仓库现有文风起草口播稿。

### 风格参考选择

优先选择文件名或主题与当前话题接近的文档。常见信号包括：
- `视频日记`
- `短视频`
- `AI时代`
- `个人竞争力`
- `数字分身`
- `知识管理`
- `表达`

不要把整个目录都读一遍，只挑最相关的少量文件。

### 文字稿要求

口播稿通常应当：
- 从一个明确的观点、判断或顿悟切入
- 展开当天真实发生的事件、例子或小故事
- 最后落到情绪、反思或留白式收尾

默认结构：

1. 开场日期或场景
2. 核心观点
3. 来自所选流水账条目的故事或论据
4. 反思与情绪收尾

默认语气：
- 第一人称
- 口语化、自然
- 有反思感，但不过度表演
- 基于真实的工作和生活细节

合适时，显式采用“观点 + 故事 + 情感”的结构。不要把内容写成泛泛的鸡汤或空泛励志文案。

## 能力二：文字稿生成候选标题

生成标题前，先读取 [title-principles.md](references/title-principles.md)。

按下面的流程执行：

1. 读取文字稿或文章草稿。
2. 提炼真正的标题抓手：
   - 共性问题
   - 最强场景
   - 最出人意料的转折
   - 情绪张力
3. 用不同风格生成多组候选标题。

### 标题输出默认规则

- 默认生成 10 个标题，除非用户指定其他数量。
- 每个标题尽量简洁，最好控制在 30 个汉字以内。
- 每个标题前标注类型标签。
- 避免大量重复同一种句式。

### 标题质量要求

优先生成这样的标题：
- 像是受众会想点开的内容，而不是作者自己在总结
- 能提出一个共性问题，而不是平铺直叙概括内容
- 包含场景、故事、反差、数字或好奇心缺口
- 不要过早把答案全部讲完

避免生成这样的标题：
- 像提纲
- 像作者自言自语
- 只有大道理，没有场景和冲突

## 输出要求

如果用户只要口播稿，只返回口播稿。
如果用户只要标题，只返回标题。
如果用户两个都要，先返回口播稿，再返回标题。

如果用户要求写回仓库，就按用户指定路径落盘；否则默认只在对话里返回结果。
