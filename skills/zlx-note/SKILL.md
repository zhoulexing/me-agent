## 工作流程

### 1. 基本信息管理

#### 查询基本信息
**触发词识别**："查询基本信息"

**操作步骤**：
1. 使用 Glob 工具列出 `$PROJECT_ROOT/01-基本信息/` 目录文件
2. 根据查询主题选择相关文档
3. 读取文档内容
4. 返回基本信息

#### 记录基本信息
**触发词识别**："记录基本信息"、"更新基本信息"

**操作步骤**：
1. 确认信息分类（如个人资料、联系方式等）
2. 生成文件名：`[信息分类].md`
3. 创建或更新文件到 `$PROJECT_ROOT/01-基本信息/`
4. 使用标准格式记录信息

### 2. 个人认知管理

#### 查询个人认知
**触发词识别**："查询个人认知"

**操作步骤**：
1. 使用 Glob 工具列出 `$PROJECT_ROOT/02-个人认知/` 目录文件
2. 根据对话主题选择相关文档
3. 读取文档内容
4. 基于个人认知给出响应

#### 添加个人认知
**触发词识别**："添加个人认知"、"更新个人认知"

**操作步骤**：
1. 确认认知主题或分类
2. 生成文件名：`[主题名称]-[tag1,tag2]-[YYYY-MM-DD].md`
3. 创建文件到 `$PROJECT_ROOT/02-个人认知/`
4. 将原始内容全部添加到文件，不做任何修改

### 3. 技术认知管理

#### 查询技术认知
**触发词识别**："查询技术认知"

**操作步骤**：
1. 使用 Glob 工具列出 `$PROJECT_ROOT/03-技术认知/` 目录文件
2. 根据技术主题选择相关文档
3. 读取文档内容
4. 基于技术认知给出建议

#### 添加技术认知
**触发词识别**："添加技术认知"、"更新技术认知"

**操作步骤**：
1. 确认技术主题或分类
2. 生成文件名：`[技术主题]-[tag1,tag2]-[YYYY-MM-DD].md`
3. 创建文件到 `$PROJECT_ROOT/03-技术认知/`
4. 将原始内容全部添加到文件，不做任何修改

### 4. 信息记录

#### 查看信息
**触发词识别**："查看信息记录"

**操作步骤**：
1. 使用 Glob 工具列出 `$PROJECT_ROOT/05-信息记录/` 目录下的文件
2. 根据类目文件名展示信息列表
3. 如需查看具体内容，读取对应文件

#### 记录信息
**触发词识别**："记录信息"、"更新信息"

**操作步骤**：
1. 先判断这条内容属于哪个“长期类目”，同类内容尽量收纳到同一个文件
2. 生成文件名：`[类目].md`
3. 优先使用已有类目；没有合适类目时再新建
4. 创建或更新文件到 `$PROJECT_ROOT/05-信息记录/`
5. 在对应类目下按条目追加，保留记录日期、名称、来源、备注等信息
6. 同一个类目文件中，不同条目之间使用 `---` 分割线隔开，便于浏览和后续追加
7. 如不确定类目，优先按“后续怎么查找”来归类，而不是按当天日期归类
8. 素材类内容不要放到 `05-信息记录/`，转存到 `04-素材库/`
9. 不再维护 `TODO.md`；研究型待办优先归到 `技术.md`

**推荐类目示例**：
- `金句.md`：想反复查看的句子、观点、表达
- `好物清单.md`：买过觉得好、以后可能复购的东西，比如烟花、工具、设备
- `模板方法.md`：值得复用的模板、提示词、方法论、工作流
- `地点清单.md`：餐厅、露营地、景点、店铺等地点信息
- `重要备忘.md`：规则、约束、以后容易忘但必须回看的信息
- `技术.md`：技术工具、概念、方案、研究项

**模板参考**：详见 `references/templates/info-record.md`

### 5. 素材库

#### 查看素材
**触发词识别**："查看素材"

**操作步骤**：
1. 使用 Glob 工具列出 `$PROJECT_ROOT/04-素材库/` 目录下的文件
2. 根据主题文件名展示素材列表
3. 如需查看具体内容，读取对应文件

#### 添加素材
**触发词识别**："保存素材"、"添加素材"、"更新素材"

**操作步骤**：
1. 先判断这条内容属于哪个“素材主题”
2. 创建或更新文件到 `$PROJECT_ROOT/04-素材库/`
3. 在对应文件下按条目追加，保留 `时间`、`标题`、`摘要`、`源链接`
4. 同一文件中不同条目之间使用 `----` 分割
5. 本地 `04-素材库/` 要与 Get 笔记中的 `素材库` 知识库一一对应

### 6. 流水账管理

#### 查看流水账
**触发词识别**："查看流水账"

**操作步骤**：
1. 确认日期（默认当前日期，或指定日期）
2. 读取 `$PROJECT_ROOT/06-流水账/YYYY-MM-DD.md`
3. 展示完整内容

#### 添加流水账
**触发词识别**："添加到流水账"

**操作步骤**：
1. 确认日期（使用当前日期 YYYY-MM-DD）
2. 检查 `$PROJECT_ROOT/06-流水账/YYYY-MM-DD.md` 是否存在
3. 文件不存在 → 参考模版创建新文件
4. 文件已存在 → 追加内容到对应章节
5. 普通流水账条目统一按以下格式记录：
   - `时间点：当前记录时间（hh:mm，如 08:50）`
   - `标签：xx`
   - `原始内容：xx`
6. 同一个章节下，不同流水账条目之间使用 `----` 分割线隔开
7. 本地日期文件应与 Get 笔记中的 `GetSeek录音卡` 一一对应，不再额外维护素材分流文件

**模板参考**：详见 `references/templates/daily-journal.md`

### 7. 笔记同步

#### 同步到Get笔记
**触发词识别**："同步到 get笔记"

**操作步骤**：
1. 确认要同步的文件路径
2. 根据路径确定知识库：
   - `02-个人认知/xxx.md` → 知识库 `个人认知`
   - `03-技术认知/xxx.md` → 知识库 `技术认知`
   - `04-素材库/xxx.md` → 知识库 `素材库`
   - `05-信息记录/xxx.md` → 知识库 `信息记录`
   - `06-流水账/YYYY-MM-DD.md` → `GetSeek录音卡`
   - 其他同上 
3. 调用 `getnote` 技能：
   - 标题：文件名（不含扩展名如tag和时间）
   - 内容：文件正文
   - 知识库：映射后的知识库名称
4. 反馈同步结果

**映射规则**：详见 `references/paths.md`

#### Get笔记下载
**触发词识别**："Get笔记下载"

**操作步骤**：
1. 确认要下载的知识库和笔记
2. 调用 Get 笔记技能下载笔记内容
3. 根据内容确定本地路径：
   - 知识库 `个人认知` → `02-个人认知/`
   - 知识库 `技术认知` → `03-技术认知/`
   - 知识库 `素材库` → `04-素材库/`
   - 知识库 `信息记录` → `05-信息记录/`
   - `GetSeek录音卡` → `06-流水账/`
4. 创建或更新本地文件
5. 反馈下载结果
