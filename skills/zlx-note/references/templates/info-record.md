# 信息记录模板

**使用场景**：记录那些“以后还要再查、再用、再买、再引用”的信息

**目标目录**：`~/clawd/project/me-agent/05-信息记录/`

**文件命名**：`[类目].md`

**核心原则**：同类信息尽量放在同一个文件里，方便长期积累和后续检索，不再按日期拆成多个文件

**条目分隔**：同一个类目文件中，不同条目之间使用 `---` 分割线隔开

**推荐类目**：
- `金句.md`：金句、好表达、值得反复引用的句子
- `好物清单.md`：买过觉得好、以后还会回购的东西
- `模板方法.md`：模板、提示词、工作流、方法论
- `地点清单.md`：餐厅、露营地、景点、店铺等地点信息
- `待整理.md`：暂时要记住，但还没决定归类的内容

---

## 模板内容

```markdown
# [类目]

## [条目名称]

- **日期**：YYYY-MM-DD
- **类目**：
- **标签**：
- **来源**：

### 记录内容


### 备注

```

---

## 使用说明

### 条目名称
每条信息用一个二级标题区分，标题尽量直接写成后续会搜索的名字

### 条目分隔
如果文件中已经有其他条目，新增内容前先写一行 `---`，再开始新的二级标题

### 日期
记录这条信息第一次记下或本次更新的日期

### 类目
类目名通常与文件名一致，方便后续迁移或重组

### 标签
补充几个检索关键词即可，不需要太多

### 来源
可以是文章链接、购买渠道、视频、聊天记录、自己实践所得

### 记录内容
按内容类型灵活写，不要求统一成僵硬字段。重点是以后还能快速看懂和复用。

### 备注
补充个人判断、适用场景、下次行动、是否推荐回购等

## 示例

### 示例 1：金句

```markdown
# 金句

## 不快，不慢，不使劲，也不松懈

- **日期**：2026-03-25
- **类目**：金句
- **标签**：节奏、长期主义、AI时代
- **来源**：罗振宇视频日记里引用的村上春树

### 记录内容

不快，不慢，不使劲，也不松懈，以一种均衡的节奏跑下去。无论别人怎么看，不要打乱自己的节奏。

### 备注

- 适合以后写 AI 时代节奏、长期主义、自我成长类内容时引用

---

## Code is cheap, Show me the talk

- **日期**：2026-02-28
- **类目**：金句
- **标签**：名言、技术文化、表达
- **来源**：技术圈表达演变记录

### 记录内容

- 原版：Talk is cheap. Show me the code.
- 新版：Code is cheap. Show me the talk.

### 备注

- 适合用来表达 AI 时代从代码能力转向表达能力与沟通能力的变化
```

### 示例 3：模板方法

```markdown
# 模板方法

## 日记模板

- **日期**：2026-02-24
- **类目**：模板方法
- **标签**：日记、模板、复用
- **来源**：自己整理 / 外部参考后沉淀

### 记录内容

- 这个模板适合长期记录每日事件、新知、素材和碎记
- 后续写日记、做视频日记、整理数字分身素材时都可以复用

### 备注

- 这类“以后还会反复拿来用”的内容，统一放到模板方法类，不要按日期单独建文件
```
