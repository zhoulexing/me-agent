## Get 笔记知识库映射规则

### 映射逻辑

**本地文件夹 → Get 笔记知识库映射规则**：
- 去掉序号前缀（如 `02-`），保留后半部分作为知识库名
- 示例：`02-个人认知/` → 知识库 `个人认知`
- 示例：`03-技术认知/` → 知识库 `技术认知`

### 映射表

| 项目文件夹 | 知识库名称 | 说明 |
|-----------|-----------|------|
| `02-个人认知/` | `个人认知` | 世界观、价值观、思维方式 |
| `03-技术认知/` | `技术认知` | 技术栈、架构观、工程理念 |
| `05-信息记录/` | `信息记录` | 体验、文档、操作指南等 |

### 同步格式

当同步到 Get 笔记时：
- **标题**：文件名（不含扩展名，不含标签和时间）
- **内容**：文件的完整正文内容
- **知识库**：根据上述映射表确定

## 文件命名规范

### 日记文件（06-流水账/）
- 格式：`YYYY-MM-DD.md`
- 示例：`2026-03-24.md`

### 信息记录文件（05-信息记录/）
- 格式：`YYYY-MM-DD-[分类]-[名称].md`
- 分类：`experience`、`doc`、`other`
- 示例：`2026-03-24-experience-某某餐厅.md`

### 素材库文件（07-素材库/）
- 格式：`YYYY-MM-DD.md`
- 示例：`2026-03-24.md`
- 说明：同一天的素材积累在同一文件中

