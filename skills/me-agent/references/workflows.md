# me-agent 详细工作流程

## 1. 日记管理流程

### 触发词识别

**直接触发**：
- "记日记"
- "添加到流水账"
- "今天发生了什么"
- "记录一下今天"
- "流水账"

**隐式触发**：
- "今天我..."
- "记录今天的..."

### 操作步骤

#### 步骤 1：确认日期
- 使用当前日期（YYYY-MM-DD 格式）
- 时区：Asia/Shanghai

#### 步骤 2：检查文件是否存在
```bash
目标路径：~/clawd/project/me-agent/06-流水账/YYYY-MM-DD.md
```

使用 Bash 或 Read 工具检查：
- 文件存在 → 进入步骤 4（追加内容）
- 文件不存在 → 进入步骤 3（创建新文件）

#### 步骤 3：创建新文件
写入日记模板：

```markdown
# 2026-03-24

## 今日事件


## 今日书账


## 今日美食


## 今日新知


## 今日碎记

```

#### 步骤 4：追加内容
- 根据用户提供的内容，追加到对应章节
- 保持格式一致性
- 使用列表或段落形式组织内容

#### 步骤 5：确认完成
- 反馈："已添加到今日日记（06-流水账/YYYY-MM-DD.md）"

## 2. 信息记录流程

### 触发词识别

**直接触发**：
- "记一下信息"
- "保存这个"
- "帮我记录"
- "记录到信息记录"

**隐式触发**：
- "这个餐厅不错，记一下"
- "保存这个教程"
- "记录这个操作指南"

### 操作步骤

#### 步骤 1：确定分类

询问用户或根据内容判断：

| 分类 | 适用场景 | 示例 |
|-----|---------|------|
| `experience` | 体验类 | 餐厅、景点、产品体验、服务评价 |
| `doc` | 文档类 | 教程、操作指南、技术文档、学习笔记 |
| `other` | 其他 | 无法归类的信息 |

#### 步骤 2：生成文件名

格式：`YYYY-MM-DD-[分类]-[名称].md`

示例：
- `2026-03-24-experience-某某餐厅.md`
- `2026-03-24-doc-git使用教程.md`
- `2026-03-24-other-待整理信息.md`

#### 步骤 3：创建文件

目标路径：`~/clawd/project/me-agent/05-信息记录/`

使用模板（详见 `templates/info-record.md`）：

```markdown
# [名称]

## 基本信息

- **名称**：
- **分类**：experience / doc / other
- **日期**：YYYY-MM-DD
- **标签**：

## 详细信息


## 个人备注

```

#### 步骤 4：填充内容
- 根据用户提供的信息填充各个字段
- 保持结构清晰，便于后续查找

#### 步骤 5：确认完成
- 反馈："已创建信息记录（05-信息记录/YYYY-MM-DD-[分类]-[名称].md）"

## 3. 素材库管理流程

- "保存素材"
- "添加到素材库"
- "记录这个文章"

**隐式触发**：
- "这篇文章不错，保存一下"
- "这个视频很有价值"
- "帮我收集这个素材"

### 操作步骤

#### 步骤 1：确认日期
- 使用当前日期（YYYY-MM-DD）

#### 步骤 2：检查文件是否存在

目标路径：`~/clawd/project/me-agent/07-素材库/YYYY-MM-DD.md`

- 文件存在 → 进入步骤 4（追加素材）
- 文件不存在 → 进入步骤 3（创建新文件）

#### 步骤 3：创建新文件

写入文件头（可选）：
```markdown
# 2026-03-24 素材收集

```

```markdown
## [标题]

### 总结

- **来源**：公众号名称 / 网站名称 / 视频平台等
- **链接**：原始链接地址
- **日期**：YYYY-MM-DD

```

#### 步骤 5：确认完成
- 反馈："已添加到素材库（07-素材库/YYYY-MM-DD.md）"

**注意**：同一天的素材积累在同一文件中，每个素材用二级标题（`##`）分隔

## 4. Get 笔记同步流程

### 触发词识别

**直接触发**：
- "同步到 get笔记"
- "同步到 Get 笔记"
- "添加到知识库"

**隐式触发**：
- "把这个同步到笔记"
- "发布到知识库"

### 操作步骤

#### 步骤 1：确认文件路径
- 从对话中获取要同步的文件路径
- 或列出最近修改的文件供用户选择

#### 步骤 2：确定知识库

根据文件路径映射：

| 文件路径模式 | 知识库名称 |
|------------|-----------|
| `~/clawd/project/me-agent/02-个人认知/*.md` | `个人认知` |
| `~/clawd/project/me-agent/03-技术认知/*.md` | `技术认知` |
| `~/clawd/project/me-agent/05-信息记录/*.md` | `信息记录` |

#### 步骤 3：读取文件内容

使用 Read 工具读取文件全文

#### 步骤 4：调用 Get 笔记技能

使用 Skill 工具调用 `getnote`：

```python
# 参数
title = 文件名（不含扩展名）
content = 文件正文内容
knowledge_base = 映射后的知识库名称
```

**注意**：
- 标题使用文件名，不带标签和时间
- 内容使用文件的完整正文

#### 步骤 5：确认同步结果
- 反馈 Get 笔记的返回结果
- 确认是否成功添加到知识库

## 5. 认知查询流程

### 触发场景（被动触发）

根据对话内容自动判断：

| 触发场景 | 关键词/特征 | 查询目标 | 目录路径 |
|---------|-----------|---------|---------|
| 用户表达观点 | "我觉得"、"我认为"、"我的观点"、"我的看法" | 个人认知 | `02-个人认知/` |
| 讨论技术方案 | "技术方案"、"架构"、"如何实现"、"用哪个技术" | 技术认知 | `03-技术认知/` |
| 评估能力 | "我能做到吗"、"适合我吗"、"我会..." | 个人技能 | `04-个人技能/` |
| 查询基本信息 | "我的信息"、"我是谁" | 基本信息查询 | `01-基本信息/` |

### 操作步骤

#### 步骤 1：识别触发场景
- 分析用户对话内容
- 匹配触发场景特征
- 确定要查询的认知目录

#### 步骤 2：列出目录文件

使用 Glob 工具：
```bash
~/clawd/project/me-agent/02-个人认知/*.md
```

#### 步骤 3：选择相关文档

根据文件名和对话主题判断相关性：
- 文件名包含相关关键词
- 文件名可能与当前话题相关
- 优先选择最近修改的文档

#### 步骤 4：读取文档内容

使用 Read 工具读取相关文档

#### 步骤 5：基于认知给出响应

**重要**：
- **不是通用回复**：基于用户的具体认知内容给出建议
- **结合价值观**：参考 `02-个人认知/` 中的价值观
- **考虑技术栈**：参考 `03-技术认知/` 中的技术偏好
- **评估可行性**：参考 `04-个人技能/` 中的能力边界

#### 步骤 6：持续更新认知

如果用户在对话中表达了新的观点或认知变化：
- 提示："这看起来是一个新的认知观点，是否需要记录到个人认知？"
- 如用户确认，创建新的认知文档

## 注意事项

### 1. 路径正确性
- 始终使用绝对路径：`~/clawd/project/me-agent/`
- 避免使用相对路径

### 2. 文件命名规范
- 严格遵守各目录的命名约定
- 日期格式：YYYY-MM-DD
- 分类标识：experience、doc、other

### 3. 模板一致性
- 创建文件时使用对应模板
- 保持格式统一，便于后续查找和管理

### 4. 认知驱动
- 基于用户认知文档提供建议
- 避免通用化的回复
- 结合用户的价值观、技术栈、能力边界

### 5. 错误处理
- 文件创建失败时，检查目录是否存在
- 文件读取失败时，给出明确的错误提示
- Get 笔记同步失败时，记录错误并建议手动处理
