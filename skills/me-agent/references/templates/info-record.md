# 信息记录模板

**使用场景**：记录餐厅、景点、教程、操作指南等有价值的信息

**目标目录**：`~/clawd/project/me-agent/05-信息记录/`

**文件命名**：`YYYY-MM-DD-[分类]-[名称].md`

**分类**：
- `experience`（体验类）：餐厅、景点、产品体验、服务评价
- `doc`（文档类）：教程、操作指南、技术文档、学习笔记
- `other`（其他）：无法归类的信息

---

## 模板内容

```markdown
# [名称]

## 基本信息

- **名称**：
- **分类**：experience / doc / other
- **日期**：YYYY-MM-DD
- **标签**：

## 详细信息


## 个人备注

```

---

## 使用说明

### 基本信息
- **名称**：信息对象的名称
- **分类**：experience/doc/other
- **日期**：记录日期
- **标签**：便于检索的关键词（可选）

### 详细信息
根据分类记录不同的内容：

**experience（体验类）**：
- 体验对象的具体描述
- 体验时间、地点、人物
- 体验过程和感受
- 优缺点分析
- 推荐指数

**doc（文档类）**：
- 文档的主题和目标
- 关键步骤和要点
- 代码示例（如有）
- 注意事项和坑点
- 适用场景

**other（其他）**：
- 信息内容描述
- 来源和背景
- 个人理解和思考

### 个人备注
- 待补充的内容
- 后续行动计划
- 关联的其他信息

### 示例 2：文档类（教程）

```markdown
# Git Rebase 使用教程

## 基本信息

- **名称**：Git Rebase 使用教程
- **分类**：doc
- **日期**：2026-03-24
- **标签**：git、版本控制、教程

## 详细信息

**用途**：将一个分支的修改整合到另一个分支，保持提交历史的线性

**基本用法**：
```bash
# 将 feature 分支 rebase 到 main
git checkout feature
git rebase main
```

**交互式 rebase**：
```bash
# 修改最近 3 次提交
git rebase -i HEAD~3
```

**注意事项**：
- 不要在公共分支上使用 rebase
- rebase 会改写历史，谨慎操作
- 如果冲突，解决后使用 `git rebase --continue`

**适用场景**：
- 合并分支前保持历史整洁
- 修改最近的提交信息
- 合并多个提交

## 个人备注

- 需要多练习交互式 rebase
- 记得在 rebase 前创建备份分支
```
