# 论文笔记：Single-Agent with Skills 何时替代 Multi-Agent Systems

- **日期**：2026-04-16
- **原文**：https://arxiv.org/pdf/2601.04748
- **标签**：AI Agent, Multi-Agent, Skill-Based, 论文笔记

---

## 一句话总结

多智能体系统（MAS）可以被编译成单智能体+技能库（SAS），在保持精度的同时减少 54% token 使用和 50% 延迟；但技能库规模扩大后，技能选择准确性会呈现**相变临界点**式的急剧下降，而非线性衰减。

---

## 核心贡献

### 1. 证明 MAS → SAS 编译可行且高效

多智能体系统编译成单智能体+技能库后：
- **精度损失**：−2% ~ +4%（平均 +0.7%，SAS 有时更强）
- **Token 节省**：53.7%（最高 58.4%）
- **延迟降低**：49.5%（最高 60.9%）
- **API 调用**：3-4 次 → 1 次

### 2. 发现技能选择的Scaling Law（相变现象）

通过认知科学框架（Simon 的有限理性）提出：LLM 技能选择存在**容量阈值 κ**：

- 当技能库规模 |S| < κ 时：选择准确率稳定在高位
- 当 |S| > κ 时：准确率急剧下降，出现**相变**
- 下降驱动因素是**语义混淆度**（I(S)），而非单纯库规模

公式：
```
ACC(σ, S) ≈ α / (1 + (|S|/κ)^γ) − ε · I(S)
```

### 3. 分层路由可缓解规模问题

当扁平选择失效时，引入**分层路由**（先粗选类别，再细选技能），类似人类的 chunking 机制，可恢复选择可靠性。

| Debate / Adversarial | ❌ |
| Parallel Sampling（独立采样） | ❌ |
| Private Information（私有状态） | ❌ |

---

## 关键认知科学理论基础

1. **Hick's Law**：选择反应时间随选项数对数增长，但超过约 8 个选项后失效（∼3 bits）
2. **Miller 的魔法数字 7**：工作记忆容量有限，超出后性能急剧下降
3. **Shepard 的泛化律**：混淆概率随心理距离指数衰减（ACT-R fan effect）
4. **Chunking 理论**：专家通过分层组织将复杂选择压缩为可管理的块

---

## 实践意义

- **小规模技能库**（< 10）：直接用 SAS，效率高
- **大规模技能库**（> 50+）：用分层路由，先分类再选技能
- **技能设计**：语义可区分性比数量更重要；相似技能会互相干扰选择

---

## 核心洞察

> "技能选择的关键瓶颈不是库大小，而是**语义可区分性**。"

这篇论文从认知科学角度解释了为什么 Skill-Based Agent 在规模扩大时会失效，并提供了层级路由这一解决方案，对设计可扩展的 Agent 系统有重要参考价值。
